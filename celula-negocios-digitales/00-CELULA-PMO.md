# Célula I+D — Negocios Digitales (AISLADA)

> **Qué es esto:** una **célula nueva y aislada** del resto del estudio (no es el sector Agencia
> Digital, no es el ERP). Es un equipo propio de **consultores + expertos** con un **PMO** por encima,
> montado para una sola misión: **traer negocios digitales concretos, construibles y facturables**.
>
> **Aislamiento:** vive fuera de `docs/sectores/`. **No toca prod, ni Neon, ni pushea nada.** Todo es
> **local** hasta nueva orden del dueño ("luego vemos"). Investigación pura + propuestas ejecutables.

- **Abierta:** 2026-07-06 · **PMO:** sesión autónoma · **Estado:** equipo desplegado

---

## 1. Misión

Actuar como una **consultora que trae negocios listos para hacer reales**: productos/servicios
digitales que el estudio pueda **fabricar rápido con Claude Code**, y luego **diseñar, brandear,
marketear y cobrar**. No teoría — ideas con mercado, competencia, precios, esfuerzo y forma de cobro.

**Regla de oro:** cada idea tiene que poder construirse con Claude Code en **semanas, no años**, con un
equipo chico, y tener un **camino claro al primer peso facturado**.

## 2. Capacidades del estudio a apalancar (genéricas)

- **Fabricación de software con Claude Code** — full-stack (Next.js/TS), webapps, APIs, agentes IA.
- **Diseño & branding** — sistema de diseño, identidad, empaque de producto.
- **Marketing digital** — SEO, ads, contenido, redes.
- **Cobro / billing** — Mercado Pago (AR), tarjetas/Stripe/Lemon Squeezy (global), suscripciones.

> La célula está **aislada** de las otras áreas: no asume el ERP ni los activos de Agencia. Trata al
> estudio como una **capacidad genérica** de construir + diseñar + vender + cobrar.

## 4. Método (definición de terminado)

- **Research web REAL con fuentes citadas** — nada de memoria. Mercado, competencia, precios reales,
  tendencias frescas.
- Cada consultor entrega **2–3 propuestas concretas** con la plantilla estándar (§5) + una sección
  **"descartados con criterio"** + fuentes.
- El PMO integra en un **portfolio priorizado** con matriz de scoring y una recomendación de
  **"qué construir primero"**.
- **Sin publicar:** nada de push/deploy/DB. Todo queda local en esta carpeta.

## 5. Plantilla estándar por propuesta

```
### [Nombre] — one-liner
- Qué es / problema que resuelve
- Cómo se construye con Claude Code (alcance, stack, tiempo a MVP)
- Diseño & branding (cómo se empaqueta/marca)
- Marketing / canal de adquisición (primeros clientes)
- Cómo se cobra (modelo, pasarela, pricing de referencia con fuente)
- Mercado / demanda (tamaño, tendencia, competencia — con fuentes)
- Apalancamiento del estudio (ventaja)
- Esfuerzo a primer peso + riesgos
- Veredicto (1–10) y por qué
```
